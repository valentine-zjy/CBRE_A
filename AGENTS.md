# CBRE 商业地产 Agent 平台项目

## 项目定位
本项目面向客户售前展示，定义一套服务商业地产场景的 Agent 产品：

- 产品形态：`统一平台 + 6 类业务 Agent`
- 应用范围：商业地产，不覆盖纯住宅 C 端导流
- 表达方式：用业务任务、结果物、证据链和人工复核讲产品，不用抽象 AI 口号讲产品

本项目采用 harness 思路推进：

- `Agent != 单次问答`
- `Agent = Model + Harness`
- Harness 负责：任务编排、状态管理、工具调用、知识/文件读写、人工确认点、日志追踪、评测与验收

## 目标客户与业务范围
- 目标客户：CBRE 及同类商业地产机构的研究、交易、招商租赁、估值资管、楼宇运营、顾问与项目团队
- 优先覆盖：研究分析、交易推进、招商租赁、估值资管、楼宇运营、文档报告自动化
- 当前目标：先把售前叙事、平台能力、页面结构和演示规格固化，再进入 PoC 设计

## 六类业务 Agent
| Agent | 服务对象 | 代表任务 | 代表结果 |
| --- | --- | --- | --- |
| 研究 Agent | 研究分析师、顾问 | 市场扫描、可比项目整理 | 市场摘要、可比项目清单 |
| 交易 Agent | 交易经理、投资分析师 | 管线跟踪、尽调推进 | 状态看板、阻塞项清单 |
| 招商租赁 Agent | 招商经理、租赁经理 | 线索分级、租约推进 | 优先级列表、跟进计划 |
| 估值资管 Agent | 估值经理、资管经理 | 假设整理、情景分析 | 参数摘要、情景对比 |
| 楼宇运营 Agent | 物业经理、设施经理 | 工单协同、异常升级 | 工单队列、异常告警 |
| 文档报告 Agent | 分析师、项目经理 | 报告初稿、引用整理 | 结构化初稿、引用清单 |

## 统一平台能力
统一平台只定义到 L1 能力层，固定为 8 类：

`orchestration` `knowledge_rag` `doc_understanding` `tool_use` `multi_agent` `state_tracking` `governance` `eval_observability`

这些能力负责让 6 类业务 Agent 具备：

- 可编排
- 可连接
- 可追溯
- 可复核
- 可评测

## 协作原则
1. 先讲业务任务，再讲 Agent，再讲平台能力。
2. 所有事实与推断分开写，缺失信息标注 `未公开` 或 `待客户确认`。
3. 所有输出尽量落文件，所有能力都要可测。
4. 所有结果默认是建议态或初稿态，必须保留人工复核。
5. 第一版技术栈只写能力层，不提前承诺具体框架。
