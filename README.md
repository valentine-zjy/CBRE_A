# CBRE 商业地产 Agent 平台

这是一套面向商业地产业务的 `统一平台 + 6 类业务 Agent` 产品定义仓库，当前以客户售前展示为主，后续延伸到 PoC 和可运行 harness。

## 为什么不是普通问答
这里要做的不是“一个会聊天的 AI”，而是能围绕真实业务任务交付结果物的 Agent 系统：

- 它要处理任务链，而不是只回答一句话。
- 它要产出清单、摘要、初稿、告警、状态看板等业务结果物。
- 它要能展示证据、人工复核、权限边界和验收口径。

## 六类业务 Agent
| Agent | 解决什么问题 | 代表结果物 |
| --- | --- | --- |
| 研究 Agent | 把分散市场资料和可比项整理成可用研究输入 | 市场摘要、可比项目清单 |
| 交易 Agent | 把交易管线、尽调和审批前材料组织成可推进状态 | 状态看板、阻塞项清单 |
| 招商租赁 Agent | 把线索、房源和租约节点组织成可跟进流程 | 线索优先级、跟进计划 |
| 估值资管 Agent | 把参数、假设、情景和经营结果组织成可复核分析 | 参数摘要、情景对比 |
| 楼宇运营 Agent | 把工单、异常和服务请求组织成闭环协同 | 工单队列、异常告警 |
| 文档报告 Agent | 把多源数据和材料变成结构化初稿与引用清单 | 结构化初稿、引用清单 |

## 统一平台能力
这 6 类 Agent 共用同一套平台底座：

- `orchestration`：任务编排
- `knowledge_rag`：知识检索 / RAG
- `doc_understanding`：文档理解与结构化抽取
- `tool_use`：工具调用与系统连接
- `multi_agent`：多 Agent 协作
- `state_tracking`：状态管理与结果追踪
- `governance`：权限、审计与人工复核
- `eval_observability`：Eval 与可观测
