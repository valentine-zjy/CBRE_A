# 03 能力矩阵

## 文档目的
本文件用于将 `docs/02_competitor_fact_table.md` 中沉淀的结构化事实，映射为统一的能力维度与评分框架。
本文件只承载能力定义、评分口径、模板结构与结论模板，不预填真实产品评分。

## 使用规则
- 评分必须基于 `docs/02` 中已记录的公开事实，不得脱离证据单独打分。
- 单个产品或方案占一行，避免把多个产品合并评分。
- 每个评分都应能追溯到对应事实字段与证据来源。
- 无公开证据时必须打 `0`，不得主观补分。
- 如存在研究判断但证据不足，应在备注中标注为“研究假设”，并与正式评分依据分开。
- 本文件用于统一比较，不在此处展开产品方案、技术架构或采购建议。

## 能力维度定义
- 信息检索与问答：是否支持面向业务资料、知识库或系统信息的检索、问答与导航。
- 文档抽取与结构化：是否支持从合同、报告、表格、邮件或扫描件中提取字段并结构化输出。
- 市场研究摘要：是否支持对市场资料进行摘要、归纳、对比与研究辅助。
- 可比项目检索：是否支持查找、筛选和组织可比项目、可比资产或可比交易案例。
- 招商租赁支持：是否支持招商线索、租户匹配、租赁沟通、租约信息整理等场景。
- 交易流程辅助：是否支持交易推进中的信息汇总、流程协同、材料整理与关键节点辅助。
- 估值/投资分析辅助：是否支持估值、投资测算、假设整理、财务分析辅助等场景。
- 楼宇/设施运营支持：是否支持物业、设施、工单、运营监控或楼宇服务相关场景。
- 报告生成：是否支持研究报告、项目简报、投资备忘录、运营汇报等文档生成。
- 系统集成：是否支持 API、数据连接、第三方系统对接或多源数据整合。
- 权限与审计：是否支持权限控制、角色隔离、操作留痕、审计与合规要求。
- 本地化支持：是否支持本地语言、区域法规口径、市场习惯和本地业务表达。

## 评分说明
- `0 = 无公开证据`
- `1 = 单点功能`
- `2 = 基础模块`
- `3 = 形成业务流程支持`
- `4 = 跨流程能力`
- `5 = 平台级 / Agent化能力`

评分使用说明：
- 单项评分以公开证据能支撑的最高等级为准。
- 若只看到概念描述、营销表述或模糊介绍，默认按较低等级处理。
- 若能力跨多个场景出现，但未形成明确流程支撑，不应直接评为 `4` 或 `5`。
- `5` 分仅用于具备平台化、可扩展、跨流程协同或明显 Agent 化特征的公开能力。
