# 00 Project Charter

## 项目目标
为 CBRE 相关商业地产业务设计并逐步实现一套 Agent 系统，优先提升研究、交易、招商租赁、估值/资管、物业/设施管理、项目管理、楼宇运营与文档自动化效率。

## 范围
- 商业地产场景
- 交易/投资分析
- 招商租赁
- 估值/资管
- 物业/设施管理
- 项目管理
- 研究报告与文档自动化
- 楼宇运营

## 非范围
- 纯住宅 C 端导流
- 本轮不实现 worker、orchestrator、eval
- 本轮不接入外部服务
- 本轮不展开完整项目结构设计

## 当前阶段
- 已完成：任务文档与研究边界
- 当前动作：建立最小 harness 起点
- 后续阶段：竞品事实表、能力矩阵、CBRE业务映射、MVP设计、PoC实现、Eval与验收

## 待确认问题
- 第一批优先验证的业务流是交易分析、招商租赁还是研究报告自动化？
- `competitor_research workflow` 的最小输入输出样例是什么？
- 后续评测以文档质量、任务完成率还是人工审核效率为主？
