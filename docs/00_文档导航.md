## 三个固定文档接口

### 1. 业务 Agent 接口
后续所有业务 Agent 描述统一使用以下字段：

`agent_name` `target_role` `business_tasks[]` `inputs[]` `agent_actions[]` `outputs[]` `business_value[]` `human_checkpoints[]` `non_goals[]`

### 2. 页面区块接口
后续所有页面区块描述统一使用以下字段：

`section_name` `section_goal` `mapped_agents[]` `user_message` `proof_points[]` `expected_takeaway`

### 3. 平台能力接口
统一平台能力固定为以下八类：

`orchestration` `knowledge_rag` `doc_understanding` `tool_use` `multi_agent` `state_tracking` `governance` `eval_observability`

## 使用约束
- 先讲业务任务，再讲 Agent 动作，再讲平台能力。
- 竞品、评测、治理都保留，但不得抢在产品定义之前。
- 所有“未公开”与“待客户确认”信息必须显式标注。
- 本轮技术栈只定义到 L1 能力层，不落具体框架选型。
