# 治理与可靠性规格

本文件是 Demo 的可信底座文档，不承担主叙事。它回答的是“页面上哪些控制必须出现，哪些边界绝不能越过”。

## 基线原则
- 不静默自动执行真实业务动作。
- 不输出无来源的关键结论。
- 不跳过人工复核。
- 不隐藏失败、降级、待确认项。

## 五个控制面
| 控制面 | 必须回答的问题 | 页面表现 | 失败时处理 |
| --- | --- | --- | --- |
| 权限边界 | 谁可以看、谁可以复核、哪些信息待客户确认 | 访问范围标记、披露标记 | 阻断展示或标记为受限 |
| 证据链 | 关键结论来自哪里 | 来源卡、引用片段、trace | 标记待确认，不允许直接通过 |
| 人工复核 | 谁来决定是否继续使用结果 | 复核入口、复核状态、复核意见 | 进入 `review_required` 或人工接管 |
| 失败与降级 | 出错时是否可见、可恢复 | 错误状态、降级说明、下一步动作 | 显式提示，不能伪装成成功 |
| 审计与验收 | 是否能回溯一次运行和一次评测 | trace、版本、验收状态 | 不能进入 `pass` |

## 页面强制要求
- 任一关键结果物都必须能打开证据或复核入口。
- 任一待客户确认的信息都必须带 `需客户确认` 或 `未公开` 标记。
- 任一失败或冲突都必须在动作日志或结果物层显式可见。
- 任一通过态都必须能说明“谁复核了，依据是什么”。

## 典型风险事件
| 风险事件 | 页面应如何表现 | 处理动作 |
| --- | --- | --- |
| 无来源结论 | 结果物显示“证据缺失” | 阻止通过，回到复核或补证据 |
| 越权请求 | 显示受限提示 | 不返回超范围结果 |
| 关键字段缺失 | 显示缺失项和影响范围 | 进入待确认或人工补齐 |
| 运行失败 | 显示失败或降级状态 | 允许重试、降级或人工接管 |
| 复核驳回 | 显示驳回原因与版本差异 | 回到草稿态重新处理 |
