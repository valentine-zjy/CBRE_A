# 演示评测指标

本文件定义 Demo 里要展示的业务可信度指标。核心不是“模型评分”，而是客户能不能相信这套 Agent 可以进入 PoC。

## 四个核心指标
| 指标 | 要回答的问题 | 页面展示方式 | 建议口径 |
| --- | --- | --- | --- |
| 节省时间 | 任务从输入到结果物是否明显更快 | 前后步骤对比、节省时间卡片 | 用样例工时或步骤数对比 |
| 减少漏项 | 关键字段、阻塞项、异常是否被识别出来 | 缺失项提醒、阻塞项清单、异常告警 | 用检查清单覆盖率表示 |
| 输出可追溯 | 关键结论是否能打开证据链 | 引用卡、证据列表、trace | 用证据覆盖率表示 |
| 可复核 | 输出是否能提交、驳回、接管 | 复核状态、复核人、复核意见 | 用复核闭环率表示 |

## 支撑指标
| 指标 | 作用 |
| --- | --- |
| 权限边界清晰度 | 证明 Agent 不会越权读取或自动执行 |
| 失败可见性 | 证明任务失败、降级、待确认项都能被看到 |
| 验收状态 | 证明当前结果是 `pass`、`conditional_pass` 还是 `fail` |

## 按 Agent 的指标重点
| Agent | 第一指标 | 第二指标 |
| --- | --- | --- |
| 研究 Agent | 节省时间 | 输出可追溯 |
| 交易 Agent | 减少漏项 | 可复核 |
| 招商租赁 Agent | 减少漏项 | 节省时间 |
| 估值资管 Agent | 输出可追溯 | 可复核 |
| 楼宇运营 Agent | 减少漏项 | 可复核 |
| 文档报告 Agent | 输出可追溯 | 节省时间 |

## 页面展示要求
- 在“典型任务流”里展示单任务级指标，如当前任务节省了多少步骤。
- 在“治理与可信”里展示全局指标，如证据覆盖率和复核闭环率。
- 在“落地路线”里展示进入 PoC 的最低门槛。

## 建议的通过阈值
| 指标 | Demo 建议阈值 | PoC 建议阈值 |
| --- | --- | --- |
| 节省时间 | 能清楚展示前后对比 | 至少一个场景实现稳定提升 |
| 减少漏项 | 能展示关键漏项识别 | 关键检查项覆盖率达到可接受水平 |
| 输出可追溯 | 关键结果物均可打开证据 | 高风险结论证据覆盖率接近完整 |
| 可复核 | 所有高风险结果均可提交复核 | 复核闭环稳定可执行 |

## 不通过的典型情况
- 页面能展示结果，但打不开证据。
- 页面有结果物，但没有人工确认点。
- 页面能讲提效，但不能讲清减少了什么漏项。
- 页面有评测数字，但无法回溯这些数字对应什么任务。
