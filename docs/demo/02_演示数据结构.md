# 演示数据结构

本文件定义 Demo 的统一数据接口。所有演示数据都围绕五组对象组织：

- Agent 输入
- Agent 动作日志
- 结果物
- 证据
- 人工确认点

## 顶层结构
```json
{
  "agent_run_id": "RUN-001",
  "agent_name": "交易 Agent",
  "target_role": "交易经理",
  "agent_input": {},
  "action_log": [],
  "artifacts": [],
  "evidence": [],
  "human_checkpoints": []
}
```

## 1. Agent 输入
| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `task_id` | string | 业务任务 ID |
| `scenario_name` | string | 当前演示场景 |
| `business_goal` | string | 当前任务目标 |
| `input_sources[]` | array | 输入材料或系统记录 |
| `source_refs[]` | array | 来源 ID 列表 |
| `assumptions[]` | array | 明确标注的假设 |
| `disclosure_tag` | string | `需客户确认` 或 `未公开`，仅用于涉及内部信息的字段 |

## 2. Agent 动作日志
| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `step_id` | string | 步骤 ID |
| `step_name` | string | 步骤名称 |
| `actor` | string | 当前执行角色，如 `orchestrator`、`researcher`、`reviewer` |
| `platform_capability` | string | 使用的平台能力，如 `orchestration`、`knowledge_rag` |
| `status` | string | `queued/running/completed/fallback/review_required` |
| `input_refs[]` | array | 本步骤读取的输入或上一步结果 |
| `output_refs[]` | array | 本步骤产生的结果物或证据 |
| `trace_id` | string | 本步骤 trace |
| `notes` | string | 关键说明、失败原因或待确认项 |

## 3. 结果物
| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `artifact_id` | string | 结果物 ID |
| `artifact_type` | string | `summary/list/draft/alert/checklist` |
| `title` | string | 结果物标题 |
| `owner_agent` | string | 归属 Agent |
| `summary` | string | 结果物摘要 |
| `linked_evidence_ids[]` | array | 关联证据 |
| `review_status` | string | `draft/review_required/approved/rejected` |
| `version` | string | 版本号 |

## 4. 证据
| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `evidence_id` | string | 证据 ID |
| `source_type` | string | `report/system_record/email/template/public_source` |
| `source_name` | string | 来源名称 |
| `snippet` | string | 证据片段或摘要 |
| `linked_artifact_ids[]` | array | 关联结果物 |
| `confidence` | string | `high/medium/low` |
| `disclosure_tag` | string | `需客户确认` 或 `未公开`，用于内部来源 |

## 5. 人工确认点
| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `checkpoint_id` | string | 确认点 ID |
| `checkpoint_name` | string | 如“提交复核”“审批前确认” |
| `trigger_condition` | string | 何时触发 |
| `reviewer_role` | string | 复核角色 |
| `decision` | string | `pending/approved/rejected/taken_over` |
| `comments` | string | 复核意见 |

## 页面取数建议
| 页面区块 | 主要读取的数据组 |
| --- | --- |
| 六类 Agent 全景 | `agent_input`、`artifacts` |
| 典型任务流 | `action_log`、`artifacts`、`human_checkpoints` |
| 治理与可信 | `evidence`、`human_checkpoints`、`action_log.trace_id` |
