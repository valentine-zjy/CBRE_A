# 02 Competitor Fact Table

## 文档目的
本文件用于沉淀竞品研究的共享事实层，作为后续能力矩阵、业务映射、MVP 设计与方案讨论的统一输入。
本文件只记录可追溯事实、明确状态标注和结构化备注，不直接输出结论。

## 使用规则
- 一行只记录一个“公司 + 产品/方案”对象，避免在同一行混合多个产品。
- 每条记录必须填写“证据来源”；没有证据时不得写成事实。
- 无法确认的信息只能标注为“未公开”“待补充”或“研究假设”，不得留成模糊表述。
- 如同一字段存在多个公开说法，应在“备注”中说明差异，并保留最直接的证据来源。
- 本文档先做事实归档，不在此处展开优劣判断、产品方案或技术架构推导。

## 字段说明
- 公司：竞品所属公司名称。
- 产品名：公开对外使用的产品、平台或方案名称。
- 国家/市场：主要运营国家、区域或目标市场。
- 目标客户：面向的客户类型，如业主、资管方、经纪团队、设施管理团队等。
- 应用场景：产品主要解决的业务场景。
- 核心功能：公开材料中明确描述的主要功能。
- AI/Agent能力：公开披露的 AI、自动化或 Agent 相关能力。
- 数据集成能力：公开披露的数据接入、系统集成、API、平台连接能力。
- 定价模式：公开可见的定价方式，如订阅制、项目制、企业定制等。
- 发布时间线索：可用于判断发布时间或演进时间线的公开线索。
- 公开案例：公开客户案例、落地案例或合作案例。
- 证据来源：用于支撑该行信息的公开来源链接、文档名或页面标题。
- 备注：补充限制条件、版本差异、口径差异或待核实说明。

## 信息来源口径
- 优先使用公司官网、产品页、官方博客、帮助中心、开发者文档、官方新闻稿。
- 其次使用客户案例页、官方演示材料、官方社交媒体公开内容。
- 第三层可使用权威媒体、行业报告或合作伙伴页面，但需在“备注”中标明非一手来源。
- 非官方二手解读不得单独作为关键字段唯一依据。
- 同一条记录尽量保留可回溯到原始页面的来源信息。

## “未公开 / 待补充 / 研究假设”的标注规则
- 未公开：经过当前研究范围内检索，未发现公开可验证信息。
- 待补充：理论上应存在公开信息，但当前尚未完成查证或证据未整理入表。
- 研究假设：基于有限线索做出的工作假设，尚未获得公开证据确认，后续必须验证或删除。
